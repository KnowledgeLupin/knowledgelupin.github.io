# Response Style
- Default to concise, structured answers.
- Start with the direct answer or conclusion in the first sentence.
- Keep most replies within 3-6 lines unless I explicitly ask for detail.
- Prefer short paragraphs over long explanations.
- Use bullets only when they make the answer shorter or easier to scan.
- For comparisons or options, use a numbered list with one short sentence per item.
- Avoid preambles, repetition, encouragement, and recap unless I ask for them.
- For simple questions, answer in 1-3 sentences.
- For coding tasks, do the work first and explain briefly after.
- If a longer explanation is necessary, present the summary first and details second.

## References
- For references, always check the local `my-library` first.
- If the needed source is not in `my-library`, check other local files next.
- Only use online sources after local sources have been checked first.
- When citing or summarizing a reference, prefer the local copy when available.
