# 선형대수 블로그 포스팅 집필 기준

이 문서는 선형대수 블로그 초안 자동화의 `집필 기준만 발췌한 요약본`입니다.  
전체 자동화 설계, 상태 모델, 후보 선정 규칙, 게시 흐름은 [2026-04-16-linear-algebra-independent-post-automation-design.md](/Users/kwkwon/Desktop/Obsidian/Mathematician/_active/github-blog/_reference/2026-04-16-linear-algebra-independent-post-automation-design.md)를 따릅니다.

## 기본 방향
- 교과서식 제목은 유지합니다.
- 그러나 글의 내부 구조는 교과서식 고정 템플릿을 따르지 않습니다.
- 공개 글은 연재의 한 회차가 아니라 독립 포스트처럼 읽혀야 합니다.
- 독자가 실제로 개념을 배워 갈 수 있을 정도로 설명은 충분히 상세해야 합니다.
- 글은 검색엔진에 잘 포착되도록 제목, 첫 문단, 소제목, 설명문이 함께 설계되어야 합니다.
- 문체는 차분하고 설명적인 해설 톤을 유지합니다.
- `정의`, `정리`, `증명` 밖의 설명 문단은 경어체를 유지합니다.
- 설명 문단에 반말이나 거친 평서체가 섞이면 안 됩니다.
- 과장된 홍보 문장, 강의 공지문 같은 말투, 체크리스트식 나열은 피합니다.
- 공리나 조건을 소개할 때는 키워드만 나열하지 말고, 각 조건의 의미를 수식과 짧은 해설로 분명히 적습니다.

## 기대하는 독서 경험
- 글을 읽으면 “무엇을 말하려는지”가 첫 몇 문단 안에 분명해져야 합니다.
- 개념 소개에서 끝나지 않고, 왜 필요한지와 어디에 쓰이는지가 자연스럽게 연결되어야 합니다.
- 정의나 계산법을 단순 나열하지 말고, 어떤 문제를 정리해 주는지 설명해야 합니다.
- 앞선 글을 읽지 않았더라도 지금 글만으로 완결된 이해가 가능해야 합니다.

## 글의 기본 형태
- 제목 아래에서 바로 글을 시작합니다.
- 서두는 2-4문단 정도로 두고, 문제의식과 오늘의 중심 질문을 먼저 던집니다.
- 본문은 대체로 3-5개의 자연스러운 소제목으로 구성합니다.
- 소제목은 `학습목표`, `선수개념`, `자가진단퀴즈` 같은 고정식 표제가 아니라, 실제 개념 이름이나 질문형 흐름을 따라갑니다.
- 끝부분에는 `마무리`나 `결론`처럼 단순한 표제를 두어 핵심과 다음 연결을 짚습니다.

## 검색 가시성 기준
- 제목은 정확한 개념명을 유지하되, 실제 검색될 한국어 표현을 가능하면 직접 포함합니다.
- 제목이 지나치게 추상적이면 안 됩니다.
- 첫 두 문단 안에는 글의 핵심 검색어가 자연스럽게 등장해야 합니다.
- 적어도 한 개의 소제목에도 핵심 검색어가 직접 들어가는 것이 좋습니다.
- `description`은 제목 반복이 아니라, 글이 무엇을 해결해 주는지 보여 주는 한 문장이어야 합니다.
- `excerpt`는 첫 문단 복사가 아니라 검색 결과와 공유 미리보기에서 읽힐 요약문이어야 합니다.
- 태그는 분류용 개념명만이 아니라 독자가 실제로 검색할 표현도 포함해야 합니다.
- 클릭 유도형 과장 제목은 금지합니다.

## 반드시 들어가야 하는 것
- 핵심 개념의 정확한 설명
- 핵심 정리나 명제의 분명한 제시
- 적어도 하나의 직접 증명
- 작은 예제를 직접 전개하며 구조를 확인하는 계산
- 현실적인 응용 예시
- 독자가 놓치기 쉬운 지점이 있다면 별도 고정 표제 없이 자연스럽게 짚는 설명
- 본문 이해에 필요한 배경의 간략한 재설명

## 피해야 하는 것
- `도입 설명`, `학습목표`, `선수개념 체크`, `자가진단퀴즈` 같은 교재 템플릿 표제
- `손으로 따라가는 계산`, `응용으로 보는 이유`, `자주 헷갈리는 점`, `정리하며` 같은 반복적 고정 표제
- 지나치게 짧은 개요형 요약문
- 정의를 말한 뒤 바로 끝나는 얕은 설명
- 설명 없는 기호 남발
- 너무 많은 bullet 위주의 기계적 문장 배열

## 자동화 초안의 수준
- 자동화 초안은 가능한 한 `거의 바로 게시 가능한 수준`을 목표로 합니다.
- 단순 outline이 아니라, 실제 문단과 연결 문장, 증명, 예시, 응용 설명이 이미 갖춰져 있어야 합니다.
- 각 글은 독립 포스트로 읽혀야 하며, 연재 번호나 회차 감각을 드러내지 않습니다.
- 자동화는 초안 생성 뒤 Claude Code 검토만을 세 차례 반복해 문장 흐름, 문체 일관성, 소제목 구조, 마무리 단락, 그리고 ChatGPT스럽게 들리는 어색한 표현을 다듬습니다.
- 최종 문장 교정 단계에서는 Codex가 문장을 수동으로 다시 쓰지 않습니다. Claude Code 검토가 실패하면 그 사실을 기록하고 사람이 후속 지시를 내릴 때까지 초안을 그대로 둡니다.
- 자동화 초안은 본문 품질뿐 아니라 제목, `description`, `excerpt`, 태그까지 게시 가능한 수준을 목표로 합니다.

## 수학 표현 규칙
- 수식은 필요한 경우에만 넣고, 설명을 대신하게 두지 않습니다.
- 블록 수식은 `$$...$$`를 사용합니다.
- 영문 용어는 첫 등장에만 병기하고, 이후에는 문맥상 자연스러운 쪽을 택합니다.
